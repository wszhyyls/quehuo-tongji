# 错误 3706 解决方案

## 错误信息
```
运行时错误 '3706'
未找到提供程序。该程序可能未正确安装。
```

## 原因
Excel 缺少 SQL Server 数据库连接组件（ADO Provider），常见情况：
1. Office 是 32 位，但安装了 64 位的 SQL Native Client
2. 根本没有安装任何 SQL Server 连接驱动
3. 使用了不存在的 Provider 名称

## 解决方案（按推荐顺序）

### 方案1：使用 ODBC 连接（最快，无需额外安装）

Windows 自带 ODBC 驱动，修改连接字符串即可：

```vba
' 旧代码（出错）
conn.Open "Provider=SQLOLEDB;Server=..."

' 新代码（使用 ODBC）
conn.Open "Driver={ODBC Driver 17 for SQL Server};Server=121.229.175.49,1290;Database=RQZT;UID=zhyy02;PWD=密码;"
```

已在 `回写实际订货_兼容版.bas` 中实现自动降级。

### 方案2：安装 SQL Server 驱动

下载并安装以下任一驱动：

**推荐：Microsoft OLE DB Driver**
- 下载地址：https://docs.microsoft.com/zh-cn/sql/connect/oledb/download-oledb-driver-for-sql-server
- 安装后重启 Excel

**备用：ODBC Driver 17 for SQL Server**
- 下载地址：https://docs.microsoft.com/zh-cn/sql/connect/odbc/download-odbc-driver-for-sql-server

### 方案3：检查 Office 位数

1. Excel → 文件 → 账户 → 关于 Excel
2. 查看是 32 位还是 64 位
3. 安装对应位数的驱动（32位Office需32位驱动）

## 快速修复步骤

1. **直接使用兼容版代码**
   - 打开 `回写实际订货_兼容版.bas`
   - 导入到 Excel VBA 编辑器
   - 修改密码后直接运行

2. **如果还是报错**
   - 打开 `控制面板 → 管理工具 → ODBC 数据源`
   - 检查是否有 "ODBC Driver 17 for SQL Server"
   - 如果没有，下载安装 ODBC Driver

## 修改后的代码重点

```vba
Private Function GetConnection() As Object
    Dim conn As Object
    Set conn = CreateObject("ADODB.Connection")
    
    ' 依次尝试5种连接方式
    On Error Resume Next
    
    ' 方式1: MSOLEDBSQL（最新）
    conn.Open "Provider=MSOLEDBSQL;Server=..."
    If Err = 0 Then GoTo Success
    
    ' 方式2: SQLOLEDB（旧版）
    conn.Open "Provider=SQLOLEDB;Server=..."
    If Err = 0 Then GoTo Success
    
    ' 方式3: ODBC 17（推荐兼容）
    conn.Open "Driver={ODBC Driver 17 for SQL Server};Server=..."
    If Err = 0 Then GoTo Success
    
    ' ...更多降级方案
    
Success:
    Set GetConnection = conn
End Function
```
