# WSZH-ShortageStore 自动更新配置指南

## 功能说明

门店端桌面软件支持**自动检测更新**和**一键更新**，无需手动下载安装包。

---

## 自动更新流程

```
┌─────────────────────────────────────────┐
│  1. 用户启动软件                         │
│          ↓                               │
│  2. 启动 3 秒后自动检查更新              │
│          ↓                               │
│  3. 发现新版本 → 自动下载                │
│          ↓                               │
│  4. 下载完成 → 提示用户                  │
│          ↓                               │
│  5. 用户点击"立即更新" → 重启安装       │
└─────────────────────────────────────────┘
```

---

## 配置步骤

### 第一步：部署更新检查函数

1. 打开 Supabase Dashboard → **SQL Editor**
2. 创建函数或使用 Edge Function：
   - 文件位置：`supabase/functions/check-update/index.ts`
3. 部署后获取函数 URL

### 第二步：上传更新文件

1. 打包新版本后，将 `dist\WSZH-ShortageStore-3.17.0.exe` 上传到 CDN
2. 记录文件 URL

### 第三步：配置更新地址

修改 `electron-main.js` 中的 `UPDATE_CHECK_URL`：

```javascript
const UPDATE_CHECK_URL = 'https://your-supabase-project.supabase.co/functions/v1/check-update';
```

同时修改 `check-update/index.ts` 中的下载地址：

```typescript
const UPDATE_URL = 'https://your-cdn.com/updates/';
```

### 第四步：更新版本号

每次发布新版本时，修改以下文件：

| 文件 | 修改位置 |
|------|---------|
| `package.json` | `"version": "3.17.0"` → `"3.18.0"` |
| `electron-main.js` | `title: 'WSZH-ShortageStore v3.18'` |
| `supabase/functions/check-update/index.ts` | `LATEST_VERSION = '3.18.0'` |

## 更新检查接口

### 请求格式

```javascript
POST https://your-project.supabase.co/functions/v1/check-update
Content-Type: application/json

{
  "version": "3.17.0"
}
```

### 响应格式

```json
{
  "success": true,
  "data": {
    "version": "3.18.0",
    "updateAvailable": true,
    "releaseDate": "2026-05-18",
    "releaseNotes": "新版本更新内容...",
    "downloadUrl": "https://cdn.com/WSZH-ShortageStore-3.18.0.exe",
    "forceUpdate": false
  }
}
```

## 发布新版本步骤

1. **修改版本号**
   ```bash
   # package.json
   "version": "3.18.0"
   
   # electron-main.js
   title: 'WSZH-ShortageStore v3.18'
   
   # supabase/functions/check-update/index.ts
   LATEST_VERSION = '3.18.0'
   ```

2. **打包**
   ```bash
   npm run build:win
   ```

3. **上传更新文件到 CDN**
   - 将 `dist\WSZH-ShortageStore-3.18.0.exe` 上传

4. **部署 check-update 函数**
   ```bash
   cd supabase
   npx supabase functions deploy check-update
   ```

5. **通知门店更新**
   - 门店端会自动检测到新版本
   - 点击"立即更新"即可

## 技术细节

### 使用 electron-updater

- 启动后 3 秒自动检查更新
- 自动下载更新包到临时目录
- 下载完成后提示用户安装
- 安装时自动退出并重启

### 安全机制

- 使用 contextIsolation 隔离渲染进程
- 通过 preload.js 安全暴露 API
- 验证更新文件签名

### 网络要求

- 需要访问 Supabase Edge Function
- 需要访问 CDN 下载更新文件

## 常见问题

### Q: 更新检查失败怎么办？

检查：
1. Supabase Edge Function 是否已部署
2. 网络是否能访问 Supabase
3. `UPDATE_CHECK_URL` 是否正确

### Q: 下载更新失败？

检查：
1. CDN 地址是否正确
2. 更新文件是否已上传
3. 网络是否稳定

### Q: 如何强制所有用户更新？

在 `check-update/index.ts` 中设置：
```typescript
forceUpdate: true
```
